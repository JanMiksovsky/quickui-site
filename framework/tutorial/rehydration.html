Core idea: For search engine visibility, controls can be created from static HTML pages created by any means (an HTML authoring tool, a CMS, etc.) by marking up HTML tags and then “rehydrated” them to create live controls.

Search engines only crawl static HTML content, not live JavaScript controls. For pages which are generally document-like, you can achieve SEO (“Search Engine Optimization”) benefits by writing HTML in normal means and including extra information that can be used later to quickly create a corresponding collection of live controls. This process is called “rehydration”, and is accomplished with a QuickUI function called Control.rehydrate().

The rehydrate() function looks for any tags with the data- property “data-control”. The value of this property should be the string name of a QuickUI control class. Other data- properties on the tag will be invoked as setters on the control. E.g., in the demo, data-paragraphs=”3” will invoke paragraphs(3) on the newly-created LoremIpsum control.

Any content within a static HTML tag will be passed to the new control’s content() setter. If you want to create complex properties in static HTML, create a child and set its data-property property to the string name of the property you want to set. E.g., in the demo, the first child with data-property=”heading” will have its HTML contents passed to the control’s heading() setter.

The demo code populates an element with a set of static HTML div elements, then invokes the QuickUI rehydrate() function to turn these into live controls. For demo purposes, the static elements are added through JavaScript; in normal practice, you would author the static

elements in whatever manner you currently used to create HTML content: an HTML editor, a content management system, etc.
