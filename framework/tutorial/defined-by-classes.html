Core idea: QuickUI controls are defined with regular JavaScript classes.

As a JavaScript developer, you already use JavaScript’s native class support to define reusable behavior. QuickUI control classes are simply JavaScript classes. There are several ways to create an instance of a control class, but the simplest is to invoke the create() method shown earlier. This function produces an instance of the given control class.

You can do anything to a control class or control instance that you can normally do in JavaScript. You can extend the class’s behavior by extending its prototype, you can test whether a control instance is a member of a given class with the JavaScript “instanceof” keyword, and so on.

The demo code creates an instance of the ColorSwatchComboBox control class by calling the class' create() method. This class provides a reasonably complex user interface element that lets a user type a CSS color name or pick one from a list of choices. The class inherits much of its behavior from superclasses called ListComboBox and ComboBox.

The demo code also adds a function to the control object that tests for membership of a class with a given name. This function is invoked several times to show that a ColorSwatchComboBox control is also an instance of some of its parent classes, but is not an instance of the BasicButton class.

Your goal: Add tests to convince yourself that a ColorSwatchComboBox is also an instance of the Control class.
