QuickUI is designed to work well with the *** CHECK LINK *** CoffeeScript language, a preprocessor for JavaScript that allows for more concise source code in many situations. In the case of QuickUI-based interfaces, CoffeeScript can let you create your user interface in less code.

Create controls in CoffeeScript

Since processed CoffeeScript is just JavaScript, all the QuickUI methods, events, etc., work as normal in CoffeeScript. For example, in CoffeeScript, you can create controls with the Control.create() method, although the CoffeeScript form can be less verbose than the equivalent JavaScript.

For example, CoffeeScript allows for concise declaration of a JavaScript dictionaries. The Control.create() method allows you to pass in such a dictionary of properties; these properties will be passed to setters on the control. For example, one of the QuickUI tutorial *** examples *** shows the construction of a CollapsibleWithHeadingButton control with JavaScript similar to the following: 

$( "body" ).append(
    CollapsibleWithHeadingButton.create()
        .heading( "News story" )
        .content([
            FlickrInterestingPhoto.create()
                .photoSize( "t" )
                .css({
                    float: "left"
                }),
            LoremIpsum.create()
        ])
);

$( "body" ).append(
    CollapsibleWithHeadingButton.create({
        heading: "News story",
        content: [
            FlickrInterestingPhoto.create({
                photoSize: "t",
                css: {
                    float: "left"
                },
            }),
            LoremIpsum.create()
        ]
    })
);

In the CoffeeScript version, most of the parenthesis, and all of the commas and curly braces, can be dropped: 

$( "body" ).append CollapsibleWithHeadingButton.create
  heading: "News story"
  content: [
    FlickrInterestingPhoto.create
      photoSize: "t"
      css:
        float: "left"
    LoremIpsum.create()
  ]

Creating QuickUI control classes in CoffeeScript

When creating QuickUI control classes in plain JavaScript, the Control.subclass() method is used to create a subclass of an existing base class. (For more information, see the *** tutorial on creating classes *** in JavaScript.) CoffeeScript makes this process easier and more concise through its built-in "class" keyword. The only extra bit unique to creating QuickUI control classes is a required a one-line boilerplate constructor: 

class MyClass extends Control
  constructor: -> return @coffee arguments

The "constructor" declaration ensures that the new class can be instantiated like any other jQuery subclass. 

# Button class showing an icon to the left of the button label.
class window.IconButton extends BasicButton
  constructor: -> return @coffee arguments
  inherited:
    content:
      "",  # *** CHECK SRC
      { html: "span", id: "IconButton_content" }
  content: Control.chain "$IconButton_content", "content"
