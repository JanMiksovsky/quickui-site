This page describes the Control JSON format which is used to by the Control.subclass() method to define a new class' property values. Control JSON is effectively a JavaScript analogue to HTML markup â€” both allow you to define the content of a user interface.

The Control JSON format allows you to concisely define two things: 1) a few fields governing your class' definition, and 2) the default property values for a new instance of the class. For example, in the following code: 

var OkButton = BasicButton.subclass({
    className: "OkButton",
    content: "OK"
});

The key "className:" is a reserved key that defines the name of the new class. The key "content:" is used to set the default value of a new instance's content() property to the text "OK".

Reserved JSON keys

There are only a handful of reserved keys at the top level of a Control JSON object:

  • className: defines the string name for the new class. This is applied as a DOM element class name to new control instances, allowing them to be styled with CSS. This name is also useful for debugging.
  • prototype: defines any members that should be added to the new class' prototype. This is where you will typically define new control properties and other methods.
  • tag: defines the HTML element type which will be used as the root for new control instances. By default, this is a div, but you can set it to another type if you wish.

Defining default property values for a new control instance

All keys at the top level of a Control JSON object are interpreted as the names of control properties. The values corresponding to these keys will be passed to the corresponding property setters. This system allows you define the default values which your new class will assign to control instances it creates.

For example, using the OkButton class definition above, the content() property for a new OkButton instance will be set, by default, to the string "OK". 

var button = OkButton.create();
var s = button.content(); // "OK"

Default values defined in the JSON passed to subclass() are actually set by invoking properties on the base class. The above definition for OkButton sets the default value by calling the content() property defined by its base class BasicButton. This allows a class to set its own default content, while simultaneously allowing the class to define a custom content() property whose definition overrides its base class' implementation. For an example of how that might work, see the tutorial exercise on Defining your control's content property.

Control JSON value types

The values in a Control JSON object can take one of the following forms:

  • String, integer, or other fundamental JavaScript type. Such a value will be used as is. Setting a control's content() property to a string representing HTML will typically place the correspoding HTML elements into the DOM.
  • An array of Control JSON objects. Each item in the array will be processed in turn.
  • A Control JSON object starting with an "html:" key. This will result in the creation of a plain jQuery object.
  • A Control JSON object starting with a "control:" key. This will create an instance of a QuickUI control of the designated class.

The latter two value types are described in more detail below.

Creating jQuery objects with the "html:" key

Suppose that you are creating a control, and somewhere within it you want to create a div using the following jQuery code: 

$( "<div/> ")
    .height( "250px" )
    .width( "300px" );

The above procedural JavaScript can be represented declaratively as data in Control JSON as follows: 

{
    html: "div",
    height: "250px",
    width: "300px"
}

The first "html:" key indicates that you want to create a new jQuery object using the specified HTML. (In normal jQuery, a single tag like "div" selects divs, rather than creating a div; the longer "<div>" is required. As a convenience, Control JSON permits the use of the shorter "div".) The "height:" and "width:" keys which follow indicate that you want to apply the height() and width() setters as a chain.

Creating QuickUI control instances with the "control:" key

Similarly, a "control:" key can be used to instantiate QuickUI controls: 

{
    control: Collapsible,
    heading: "Today's photo",
    content: {
        control: FlickrInterestingPhoto,
        photoSize: "m"
    }
}

This JSON creates a FlickrInterestingPhoto control nested within a Collapsible control. It also sets the Collapsible's heading() property, and the photo's photoSize() property.

Setting IDs on DOM elements and controls

If a Control JSON object starting with "html:" or "control:" also contains an "id:" key, the resulting element or control will have the indicated string value applied as a DOM ID. As a secondary effect, the control class being defined (not the control instance being created) will have an element reference function generated for it. This function will permit quick access to the indicated element at runtime.

See Using functions to reference elements within a control for more discussion and an interactive example.

Advantages of using JSON to define a control's appearance

The data-driven approach to using JSON to generate UI arguably has advantages over the equivalent procedural JavaScript code:

  • The JSON doesn't allow for fancy coding tricks or idioms. Such techniques can result in compact, efficient code, but at a loss of legibility. Reading someone else's jQuery code can be challenging; there's a good chance they will populate the DOM by some means other than what you yourself would have written. Control JSON, in contrast, requires you to create a UI declaratively, with static data. In the long run, this approach tends to produce UI code which is easier to read and maintain.
  • The Control JSON directly reflects the structure of the DOM it will create. With old-school HTML, it's easy to visualize the structure of the DOM the markup will produce at runtime. Similarly, the hierarchical nature of JSON makes it easy to visualize the resulting UI.
  • JSON is more parsable tool-able than arbitrary JavaScript. This enables tools which generate or transform UI code.
