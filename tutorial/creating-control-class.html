Core idea: QuickUI makes it easy for you to package up content, styling, and behavior as a reusable control class. You can do this by calling the function Control.subclass(), passing in a JSON dictionary defining default values for various class properties. The result of this call will be your new control class: a JavaScript class that you can instantiate as part of a user interface.

Control.subclass() does its core work via a jQuery method called sub(), which implements a standard JavaScript prototype-based inheritance scheme. Your class will inherit a large set of standard behavior from the base Control class. Since Control inherits from jQuery, your new class will also inherit all of jQuery’s abilities.

The JSON dictionary can contain keys reserved by the QuickUI framework. For example, the value of the reserved “className:” key will define the name for the new control class, which will be applied as a CSS class to all controls created by the class; see CSS class names. The JSON dictionary can also contain other keys. Any entry of the form foo: “bar” will cause the base class property foo() to be invoked with value “bar”.

The result of calling subclass() is typically stored in a global JavaScript variable with the same identifier as the control’s class name. It’s this form of the name that you will reference directly in your JavaScript code.

The demo code creates a new control class called Greet. This will be a subclass of the base Control class. The reserved “className:” key in the JSON dictionary sets the class’ name to “Greet”. The result of the subclass() call is stored in a JavaScript variable with the identifier Greet.

The Greet.create() calls that follow add three instances of the new Greet class to the demo. However, since Greet currently has no meaningful pre-defined content, the three controls don’t appear to be visible.

Your goal: Add default content to Greet controls by adding a “content:” key to the dictionary passed to Control.subclass(), and give this key a string value like “Hello, world!”

When Greet.create() is called, a plain instance of the base Control class is created. The string you specify will then be passed to the Control class’ content() property, which will insert the text into the control’s top element. When the resulting elements are passed to $demo.append(), you end up seeing three copies of the text.
