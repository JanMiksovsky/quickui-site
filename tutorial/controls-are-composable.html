Core idea: You can easily combine controls to create a user interface.

It’s rare to find a user interface that’s best implemented as a single, monolithic chunk of code. Rather, a typical user interface can usually be decomposed into pieces, which can themselves be decomposed, and so on. The smaller the components, the more fundamental and general in nature they tend to be.

Because QuickUI controls are generally designed to be context independent, it’s possible to recombine controls in new ways and still have the constituent parts work correctly. For example, the controls in the QuickUI Catalog all have a specific role to play, and are designed to be incorporated in novel ways to create a user interface.

The demo code creates one of the QuickUI Catalog controls, DateComboBox, which is designed to let users pick a single date by typing or navigating a dropdown month calendar. DateComboBox is in its own right, but is not delievered as a monolithic piece of functionality. Rather, it’s composed of other Catalog controls that can each be used independently or combined in new ways.

Below the DateComboBox you see some of its component parts. These aren’t wired together yet, but simply demonstrate some of the pieces you can build with to construct your own user interface elements.

  • DateTextBox, a text box that ensures its contents are a valid date in short date format. This is used for a DateComboBox’s text input portion.
  • CalendarMonthNavigator, an inline calendar that lets a user navigate by month to select a date. This is used for a DateComboBox’s dropdown.
  • MonthName, which shows the name of the month for a given date.
  • DaysOfWeek, which can show the names of the days of the week in a variety of formats.

By default these components support standard U.S. English date formats, but they are all localizable through the Globalize library.

Your goal: Add a CalendarMonth control to the end of the demo. This is the main component of a CalendarMonthNavigator, which in turn is the main component of a DateComboBox. Subsequent exercises will explain how you can begin giving controls the behavior they’d need to work together. By visually arranging and wiring together these components, it would be possible to build your own date picker control or some other date-related user interface.
